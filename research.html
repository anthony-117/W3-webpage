Difference between GET and POST in RESTful APIs

  • POST

    1. The POST method requests that the origin server accept the entity attached in the request as a new subordinate of the resource identified by the Request-URI in the Request-Line
    2. POST is NOT idempotent. So if we retry the request N times, we will end up having N resources with N different URIs created on the server.
    3. Responses to this method are not cacheable, unless the response includes appropriate Cache-Control or Expires header fields.

  • GET

    1. The GET Method requests to retrieve resource representation/information only - and not modify it in any way. As GET requests do not change the resource's state, these are said to be safe methods.
    2. GET APIs should be idempotent. Making multiple identical requests must produce the same result every time.

Google Search Algorithm

  • Algorithm

    Google Search works in three stages, and not all pages make it through each stage:

    1. Crawling: Google downloads text, images, and videos from pages it found on the internet with automated programs called crawlers.
    2. Indexing: Google analyzes the text, images, and video files on the page, and stores the information in the Google index, which is a large database.
    3. Serving search results: When a user searches on Google, Google returns information that's relevant to the user's query.

  • Page Rank

    1. Basic Concept: PageRank is based on the idea that the importance of a web page can be determined by the quantity and quality of links pointing to it from other web pages. The more high-quality websites that link to a page, the more important the page is assumed to be.
    2. Mathematical Model: PageRank can be thought of as modeling a "random surfer" who starts on a random web page and keeps clicking on links, never hitting "back" but eventually gets bored and starts on another random page. The probability that the random surfer visits a page is its PageRank.
    3. The PageRank Formula: The simplified PageRank formula for a page A is:
      PR(A) = (1-d) + d * (PR(T1)/C(T1) + ... + PR(Tn)/C(Tn))
      Where:
      • PR(A) is the PageRank of page A
      • PR(Ti) is the PageRank of pages Ti which link to page A
      • C(Ti) is the number of outbound links on page Ti
      • d is a damping factor, usually set to 0.85
    4. Damping Factor: The damping factor d represents the probability that the "random surfer" will continue clicking on links, rather than randomly jumping to a new page. The (1-d) term ensures that pages with no inbound links still get a small PageRank.
    5. Iterative Calculation: PageRank is calculated iteratively. Initially, all pages are assigned an equal PageRank, and then the formula is applied repeatedly until the PageRank values converge (i.e., stop changing significantly).
    6. Link Quality: Not all links are equal. Links from more important pages (those with higher PageRank) contribute more to the PageRank of the pages they link to.
    7. Treatment of Dangling Links: Pages with no outbound links (dangling nodes) pose a problem for the algorithm. Various methods have been proposed to handle this, such as treating them as if they linked to all pages in the web.
    8. Personalization: The basic PageRank algorithm can be modified to produce personalized PageRanks, which take into account a user's preferences or browsing history.
    9. Integration with Other Factors: While PageRank was a revolutionary algorithm when introduced, modern search engines use it in combination with hundreds of other factors to determine search rankings.

First HTML TAGS

  • Tag Description
    <title> Specifies the title of the document, shown in the browser tab or title bar.
    <nextid> An obsolete tag used to specify the next unique identifier number. It was used in early HTML versions but is no longer supported.
    <a> Defines a hyperlink, allowing users to navigate from one page or section to another using URLs.
    <isindex> An obsolete tag used to create a single-line input field for keyword searches. It is no longer used in modern HTML.
    <plaintext> An obsolete tag that causes the rest of the page to be rendered as plain text, ignoring HTML formatting. It's rarely used in current web development.
    <listing> An obsolete tag that displays text as preformatted, like <pre>, often used for code or listings.
    <p> Defines a paragraph of text. This is a block-level element, typically separating text with space above and below.
    <h1>…<h6> Defines HTML headings, with <h1> being the most important (usually the main heading) and <h6> the least important.
    <address> Represents contact information for the document author or owner, usually rendered in italics by default.
    <hp1>, <hp2> These tags are not part of standard HTML and likely refer to custom or proprietary tags in some older or non-standard HTML implementations.
    <dl> Defines a description list (also called a definition list), containing pairs of terms and descriptions.
    <dt> Defines a term or name in a description list, typically appearing in bold.
    <dd> Defines a description or value for a term in a description list, usually indented under the term.
    <ul> Defines an unordered list, with list items typically rendered as bullet points.
    <li> Defines a list item within either an unordered (<ul>) or ordered (<ol>) list.
    <menu> An obsolete tag that originally represented a list of commands or options. It was used similarly to <ul> but is now rarely used.
    <dir> Another obsolete tag that defined a directory list of files. It has been replaced by more modern elements like <ul>.

CSS History and Features

  • Creation of CSS

    Cascading Style Sheets (CSS) was proposed by Håkon Wium Lie on October 10, 1994, while working with Tim Berners-Lee at CERN. The World Wide Web Consortium (W3C) developed CSS to solve the problem of HTML becoming too complex as authors began using it for formatting instead of just structuring content.

  • CSS Levels and Versions

    CSS has evolved through several levels: CSS1 was released in 1996, CSS2 in 1998, and CSS3 began development in 1999. Unlike its predecessors, CSS3 was divided into modules that could be developed and released independently. This modular approach continues with CSS4 and beyond, though version numbers are less emphasized now.

  • CSS @rules

    @import Rule

    Imports external stylesheets or font resources from Google Fonts. 

    @import url('https://fonts.googleapis.com/css2?family=Roboto');

    @media Rule

    Creates responsive designs by applying styles based on device characteristics. 

    @media (max-width: 600px) {
        body { font-size: 14px; }
    }

    @font-face Rule

    Defines custom font resources.

    @font-face {
        font-family: 'CustomFont';
        src: url('path/to/font.woff2') format('woff2');
    }

    @keyframes Rule

    Defines animation sequences.

    @keyframes slideIn {
        from { transform: translateX(-100%); }
        to { transform: translateX(0); }
    }

    @supports Rule

    Checks browser support for specific CSS features.

    @supports (display: grid) {
        .container { 
            display: grid; 
            grid-template-columns: 1fr 1fr; 
        }
    }

    @layer Rule

    Creates cascade layers for more controlled style specificity. 

    @layer base, theme, utilities;
    @layer theme {
        body { background-color: #f0f0f0; }
    }

  • CSS Selectors

    Element Selector

    Selects all paragraphs

    p { color: #333; }

    Class Selector

    Selects elements with a specific class

    .highlight { background-color: yellow; }
    ID Selector

    Selects a unique element

    #unique-element { font-weight: bold; }
    Attribute Selector

    Selects elements with specific attributes 

    [data-type="special"] { 
        border: 2px solid blue; 
    }

    Pseudo-class Selector

    Selects elements based on a special state

    a:hover { color: red; }
    li:nth-child(even) { background-color: #f0f0f0; }

    Pseudo-element Selector

    First letter of this paragraph is styled differently

    p::first-letter {
        font-size: 2em;
        color: #ff0000;
    }

    Combinator Selectors

    Descendant Selector: Selects paragraphs inside .container

    Child Selector: Direct children
    /* Descendant Combinator */
    .container p { margin-bottom: 10px; }
    
    /* Child Combinator */
    .parent > .child { font-style: italic; }
    
    /* General Sibling Combinator */
    h2 ~ p { color: green; }
    
    /* Adjacent Sibling Combinator */
    h3 + p { margin-top: 5px; }

  • Box Model

    The CSS box model is a fundamental concept introduced early in CSS development. It describes how elements are rendered as boxes with content, padding, borders, and margins. CSS3 introduced the box-sizing property, allowing developers to change how the box model is calculated.

  • Flexbox and Grid

    CSS Flexbox (2017) and CSS Grid (2017) are modern layout systems that revolutionized web design. Flexbox provides a more efficient way to lay out, align, and distribute space among items in a container. Grid offers a two-dimensional grid-based layout system, making complex layouts more accessible and responsive design easier.

  • CSS Preprocessors

    While not part of the CSS specification, preprocessors like Sass (2006), Less (2009), and Stylus (2010) have significantly influenced CSS development. They introduced features like variables, nesting, and mixins, many of which were later adopted into native CSS.

Understanding Input vs Button Elements in HTML

The Input Tag (<input>)

The input tag is a self-closing element used to create various form controls for user input. It uses the type attribute to define its behavior.

Option 1 Option 2

Key Characteristics of <input>:

  • Self-closing tag (no end tag needed)
  • Uses type attribute to define functionality
  • Cannot contain other HTML elements
  • Value is set through attributes
  • Primarily used for collecting user input

The Button Tag (<button>)

The button tag creates a clickable button that can contain other HTML elements.

Button Examples:

Key Characteristics of <button>:

  • Requires closing tag
  • Can contain other HTML elements (text, images, icons)
  • More flexible for styling
  • Three types: submit, reset, button
  • Primarily used for user actions

Key Differences

Feature <input> <button>
Primary Purpose Collecting user input in various formats Triggering actions or form submission
Tag Structure Self-closing tag Requires closing tag
Content Value set through attributes only Can contain HTML elements and text
Flexibility Specific purpose based on type attribute General purpose, highly customizable
Common Uses Text fields, checkboxes, radio buttons, file uploads Submit buttons, action buttons, toggles

When to Use Each

Use <input> when:

  • You need to collect user input (text, numbers, dates, etc.)
  • You want to create form controls (checkboxes, radio buttons)
  • You need specialized input types (email, tel, password)
  • You want to handle file uploads

Use <button> when:

  • You need a clickable element to trigger actions
  • You want to include HTML content inside the button
  • You need more complex styling options
  • You're creating call-to-action elements

PHP Spaceship Operator (<=>)

The spaceship operator (<=>) is a comparison operator introduced in PHP 7.0. It returns:

  1. -1 if the left operand is less than the right operand
  2. 0 if the operands are equal
  3. 1 if the left operand is greater than the right operand
// Basic usage
            $result = 1 <=> 1;  // Returns 0 (equal)
            $result = 1 <=> 2;  // Returns -1 (left is less)
            $result = 2 <=> 1;  // Returns 1 (left is greater)
            
            // String comparison
            $result = "a" <=> "b";  // Returns -1
            $result = "b" <=> "b";  // Returns 0
            $result = "c" <=> "b";  // Returns 1

Common use cases include:

  1. Sorting arrays with custom comparison logic
  2. Implementing comparison methods in classes
  3. Simplifying complex conditional comparisons
// Using with usort()
            $fruits = ["apple", "banana", "cherry"];
            usort($fruits, function($a, $b) {
                return $a <=> $b;
            });
            
            // In a class comparison method
            public function compareTo($other) {
                return $this->value <=> $other->value;
            }

Static Variables in PHP

Static variables in PHP maintain their value between function calls. They are declared using the static keyword and are initialized only once when the function is first called. 

function counter() {
      static $count = 0;
      return ++$count;
  }
  
  echo counter(); // Outputs: 1
  echo counter(); // Outputs: 2
  echo counter(); // Outputs: 3

Key characteristics of static variables:

  1. They maintain their value between function invocations
  2. They are initialized only once
  3. They are only accessible within the function's scope
  4. They can be used to maintain state without using global variables

Static variables in classes:

class Logger {
      private static $logCount = 0;
      
      public function log($message) {
          self::$logCount++;
          echo "[Log #" . self::$logCount . "] $message\n";
      }
      
      public static function getLogCount() {
          return self::$logCount;
      }
  }

Common use cases:

  1. Counting function calls
  2. Caching expensive computations
  3. Implementing singleton patterns
  4. Managing shared state within a class
