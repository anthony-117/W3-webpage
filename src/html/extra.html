Difference between GET and POST in RESTful APIs

POST

  1. The POST method requests that the origin server accept the entity attached in the request as a new subordinate of the resource identified by the Request-URI in the Request-Line
  2. POST is NOT idempotent. So if we retry the request N times, we will end up having N resources with N different URIs created on the server.
  3. Responses to this method are not cacheable, unless the response includes appropriate Cache-Control or Expires header fields.

GET

  1. The GET Method requests to retrieve resource representation/information only – and not modify it in any way. As GET requests do not change the resource's state, these are said to be safe methods.
  2. GET APIs should be idempotent. Making multiple identical requests must produce the same result every time.

Google Search Algorithm

Google Search works in three stages, and not all pages make it through each stage:

  1. Crawling: Google downloads text, images, and videos from pages it found on the internet with automated programs called crawlers.
  2. Indexing: Google analyzes the text, images, and video files on the page, and stores the information in the Google index, which is a large database.
  3. Serving search results: When a user searches on Google, Google returns information that's relevant to the user's query.

Page Rank

  1. Basic Concept: PageRank is based on the idea that the importance of a web page can be determined by the quantity and quality of links pointing to it from other web pages. The more high-quality websites that link to a page, the more important the page is assumed to be.
  2. Mathematical Model: PageRank can be thought of as modeling a "random surfer" who starts on a random web page and keeps clicking on links, never hitting "back" but eventually gets bored and starts on another random page. The probability that the random surfer visits a page is its PageRank.
  3. The PageRank Formula: The simplified PageRank formula for a page A is:
    PR(A) = (1-d) + d * (PR(T1)/C(T1) + ... + PR(Tn)/C(Tn))
    Where:
    • PR(A) is the PageRank of page A
    • PR(Ti) is the PageRank of pages Ti which link to page A
    • C(Ti) is the number of outbound links on page Ti
    • d is a damping factor, usually set to 0.85
  4. Damping Factor: The damping factor d represents the probability that the "random surfer" will continue clicking on links, rather than randomly jumping to a new page. The (1-d) term ensures that pages with no inbound links still get a small PageRank.
  5. Iterative Calculation: PageRank is calculated iteratively. Initially, all pages are assigned an equal PageRank, and then the formula is applied repeatedly until the PageRank values converge (i.e., stop changing significantly).
  6. Link Quality: Not all links are equal. Links from more important pages (those with higher PageRank) contribute more to the PageRank of the pages they link to.
  7. Treatment of Dangling Links: Pages with no outbound links (dangling nodes) pose a problem for the algorithm. Various methods have been proposed to handle this, such as treating them as if they linked to all pages in the web.
  8. Personalization: The basic PageRank algorithm can be modified to produce personalized PageRanks, which take into account a user's preferences or browsing history.
  9. Integration with Other Factors: While PageRank was a revolutionary algorithm when introduced, modern search engines use it in combination with hundreds of other factors to determine search rankings.

First HTML TAGS

Tag Description
<title> Specifies the title of the document, shown in the browser tab or title bar.
<nextid> An obsolete tag used to specify the next unique identifier number. It was used in early HTML versions but is no longer supported.
<a> Defines a hyperlink, allowing users to navigate from one page or section to another using URLs.
<isindex> An obsolete tag used to create a single-line input field for keyword searches. It is no longer used in modern HTML.
<plaintext> An obsolete tag that causes the rest of the page to be rendered as plain text, ignoring HTML formatting. It's rarely used in current web development.
<listing> An obsolete tag that displays text as preformatted, like <pre>, often used for code or listings.
<p> Defines a paragraph of text. This is a block-level element, typically separating text with space above and below.
<h1>…<h6> Defines HTML headings, with <h1> being the most important (usually the main heading) and <h6> the least important.
<address> Represents contact information for the document author or owner, usually rendered in italics by default.
<hp1>, <hp2> These tags are not part of standard HTML and likely refer to custom or proprietary tags in some older or non-standard HTML implementations.
<dl> Defines a description list (also called a definition list), containing pairs of terms and descriptions.
<dt> Defines a term or name in a description list, typically appearing in bold.
<dd> Defines a description or value for a term in a description list, usually indented under the term.
<ul> Defines an unordered list, with list items typically rendered as bullet points.
<li> Defines a list item within either an unordered (<ul>) or ordered (<ol>) list.
<menu> An obsolete tag that originally represented a list of commands or options. It was used similarly to <ul> but is now rarely used.
<dir> Another obsolete tag that defined a directory list of files. It has been replaced by more modern elements like <ul>.
